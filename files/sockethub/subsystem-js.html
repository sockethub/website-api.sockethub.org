Subsystem

Handles the subscribing and listening to the redis pub/sub channel for this sockethub instance.  Used by the session class to abstract the redis specific code.

When a ping is received from any listener or dispatcher, this class will determine if the ping is targeted at it (either implicity or explicity) and if so, will emit an event.  Possible events to hook into:

Parameters

platformstring - platform name
sockethubIdstring/number - id of sockethub session

Returns

eventsan instance of the emitter class.  Use this to register ‘on’ events.
subsystem.events.on('ping', function(data) { ... });
sendused to send a command (currently only ping supported), second param is object data.
subsystem.send('ping', {encKey: 'foobar'});

Events

ping

Will emit when this platform using this session instance receives a ping.  Ping responses are handled automatically by the subsystem class, and only should be hooked into if you’d like to do something with the data object (you don’t need to reply to a ping for the sender of the ping to get a response).

Parameters

dataThe data object received.

Example data object

{
  verb: 'ping',
  actor: {
    platform: this.platform
  },
  object: {
    timestamp: '....',
    encKey: '....'
  },
  target: [
    {
      platform: 'irc'
    }
  ]
}

ping-with-callback

Same as ‘ping’, except the second parameter is the callback.  If the callback is not called, there is never a ping response.

Parameters

dataThe data object received.
callbackFunction to callback when you’re ready.  Ping response will not be issued until this callback is called.

ping-response

Triggered when the sender platform receives the response to their ping.  There is no response to this event.

Parameters

dataThe data received from the ping response.

cleanup

Triggered when a platforms receives a cleanup command issued by the dispatchers subsystem.

Parameters

dataThe data object sent, usually indicating what sessions/platforms to cleanup.

Example data object

{
  cleanup: {
   verb: 'cleanup',
   actor: {
     platform: this.platform
   },
   object: {
     timestamp: '....',
     sids: [ '...' ]
   },
   target: [
     {
       platform: 'irc'
     }
   ]
}
