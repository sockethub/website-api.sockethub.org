Listener

There is one listener per platform.  Upon initialization it subscribes to the incoming redis channel for it’s platform (sockethub:listener:email:incoming), as well as the subsystem redis channel for it’s platform (sockethub:listener:email:subsystem).

It will then reqiure the platform module and finally it waits for incoming jobs,

When it receives a job it checks for an already existing record (indexed by session id) of an instance of the session module.  If it doesn’t exist it creates one.  When it has the instance it calls the verb for that job. ie. instance.send(job, psession)
